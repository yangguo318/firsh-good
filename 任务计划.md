## Phase 2：分层渲染实现

> 目标：实现静态背景、水波纹、数字时钟的分层渲染

### 任务 2.1 — 实现 BackgroundLayer.ets
- **文件**：`entry/src/main/ets/render/layers/BackgroundLayer.ets`（新建）
- **内容**：
  - 实现 `draw(ctx, w, h)` 方法
  - 绘制水面渐变底色：`#1a4a5a` → `#0d2e3a`
  - 首次绘制后缓存为离屏 Bitmap
  - 后续帧直接 `drawImage` 从缓存取
- **验证**：背景渐变正确显示

### 任务 2.2 — 实现 WaterSurfaceLayer.ets
- **文件**：`entry/src/main/ets/render/layers/WaterSurfaceLayer.ets`（新建）
- **内容**：
  - 实现 `draw(ctx, w, h, frame)` 方法
  - 绘制水面波纹线条（半透明弧线）
  - 绘制水面光斑（小圆点）
  - 波纹随帧缓慢移动
- **验证**：水波纹可见，缓慢流动

### 任务 2.3 — 实现 ClockLayer.ets
- **文件**：`entry/src/main/ets/render/layers/ClockLayer.ets`（新建）
- **内容**：
  - 实现 `draw(ctx, w, h, now)` 方法
  - 绘制数字时间 `HH:MM:SS`，居中偏上，字号 `W * 0.12`
  - 绘制星期，字号 `W * 0.04`
  - 绘制日期 `MM/DD`，字号 `W * 0.04`
  - 无表针，纯数字显示
- **验证**：时间正确显示，位置居中偏上

### 任务 2.4 — 集成三层到 RenderEngine
- **文件**：`entry/src/main/ets/render/RenderEngine.ets`
- **内容**：
  - 在 `renderFrame()` 中按顺序调用：
    1. `BackgroundLayer.draw()`
    2. `WaterSurfaceLayer.draw()`
    3. `ClockLayer.draw()`
  - 每帧先 `clearRect` 再逐层绘制
- **验证**：三层叠加显示正确

### ~~任务 2.5 — 更新 WatchFaceAbility 集成渲染引擎~~（已取消）
> WatchFaceAbility 不存在。渲染引擎集成在 RenderEngine 内部完成，Index.ets 负责调用。

### 任务 2.6 — 从旧 Index.ets 迁移可复用绘制逻辑

### 任务 2.8 — Phase 2 集成测试

  - 背景渐变 + 水波纹 + 数字时钟三层叠加正常

## Phase 3：动画效果

> 目标：实现花瓣飘落和锦鲤游动动画

### 任务 3.1 — 实现 PetalLayer.ets 粒子系统
- **文件**：`entry/src/main/ets/render/layers/PetalLayer.ets`（新建）
- **内容**：
  - 定义花瓣池：8-12 片花瓣
  - 每片花瓣属性：位置(x,y)、速度(vx,vy)、旋转角、透明度、颜色类型
  - `init(w, h)` 初始化花瓣随机位置
  - `update(dt)` 更新花瓣位置（向下飘落 + 左右摇摆 + 旋转）
  - `draw(ctx, w, h)` 绘制花瓣（用代码绘制椭圆花瓣形状，后续替换为图片）
  - 花瓣飘出屏幕底部后重置到顶部
- **验证**：花瓣可见，向下飘落

### 任务 3.2 — 实现 KoiLayer.ets 锦鲤动画
- **文件**：`entry/src/main/ets/render/layers/KoiLayer.ets`（新建）
- **内容**：
  - 定义 3 条锦鲤（橙、红、金）
  - 每条锦鲤属性：位置、方向、速度、摆尾相位
  - `update(dt)` 更新锦鲤位置（水平游动 + 垂直正弦摆动）
  - `draw(ctx, w, h)` 绘制锦鲤（用代码绘制：椭圆身体 + 摆尾 + 眼睛 + 背鳍）
  - 锦鲤到达边界后转向
  - 从旧 `Index.ets` 迁移 `drawGoldfish` 逻辑
- **验证**：锦鲤可见，游动流畅

### 任务 3.3 — 集成花瓣层到 RenderEngine
- **文件**：`entry/src/main/ets/render/RenderEngine.ets`
- **内容**：
  - 在渲染循环中加入 `PetalLayer.update()` + `PetalLayer.draw()`
  - 顺序：背景 → 水面 → **花瓣** → 锦鲤 → 时钟
- **验证**：花瓣在水面上方、锦鲤下方正确显示

### 任务 3.4 — 集成锦鲤层到 RenderEngine
- **文件**：`entry/src/main/ets/render/RenderEngine.ets`
- **内容**：
  - 在渲染循环中加入 `KoiLayer.update()` + `KoiLayer.draw()`
  - 顺序：背景 → 水面 → 花瓣 → **锦鲤** → 时钟
- **验证**：全部5层叠加正确

### 任务 3.5 — 动画帧率调优
- **内容**：
  - 确认 20fps（50ms 间隔）渲染流畅
  - 检查是否有卡顿（特别是花瓣+锦鲤同时运动时）
  - 如卡顿，考虑降低花瓣数量或简化绘制
- **验证**：动画流畅，无明显卡顿

### 任务 3.6 — Phase 3 集成测试
- **验证**：
  - 花瓣飘落自然
  - 锦鲤游动流畅
  - 时钟正常走时
  - 所有图层叠加正确
  - 连续运行 5 分钟无崩溃

**🔵 Phase 3 验收标准**：完整花海观鱼动画效果，5层渲染流畅

## Phase 4：熄屏模式

> 目标：实现 AOD 常亮显示，支持亮屏/熄屏自动切换

### 任务 4.1 — 实现 AODLayer.ets
- **文件**：`entry/src/main/ets/render/layers/AODLayer.ets`（新建）
- **内容**：
  - 实现 `draw(ctx, w, h, now)` 方法
  - 仅绘制 `HH:MM`（不含秒），居中显示
  - 字号 `W * 0.10`，白色 40% 透明度
  - 背景全黑
- **验证**：AOD 画面正确

### 任务 4.2 — 实现显示模式切换逻辑
- **文件**：`entry/src/main/ets/render/RenderEngine.ets`
- **内容**：
  - 添加 `DisplayMode` 枚举：`Normal` / `AOD`
  - 添加 `setDisplayMode(mode)` 方法
  - `Normal` 模式：渲染全部 5 层，20fps
  - `AOD` 模式：仅渲染 `AODLayer`，1次/分钟
- **验证**：模式切换逻辑正确

### ~~任务 4.3 — 接入系统 AOD 回调~~（已取消）
> WatchFaceAbility 不存在，无系统 AOD 回调可接入。AOD 模式通过 InteractionManager.toggleAOD() 手动切换。

### 任务 4.4 — AOD 防烧屏处理
- **文件**：`entry/src/main/ets/render/layers/AODLayer.ets`
- **内容**：
  - 每分钟微调文字位置（±2px 随机偏移）
  - 使用低亮度颜色
  - 避免固定像素长时间显示
- **验证**：文字位置有微小变化

### 任务 4.5 — Phase 4 集成测试
- **验证**：
  - 亮屏时：全动画渲染
  - 熄屏时：仅显示极简时间
  - 切换过程无闪烁、无崩溃
  - AOD 模式下功耗明显降低

**🔵 Phase 4 验收标准**：亮屏/熄屏自动切换，AOD 模式正常显示

## Phase 5：交互功能

> 目标：实现触控和旋转表冠交互

### 任务 5.1 — 实现 InteractionManager.ets
- **文件**：`entry/src/main/ets/interaction/InteractionManager.ets`（新建）
- **内容**：
  - 定义交互状态枚举：`ClockStyle`（Digital/Minimal/Hidden）
  - 定义 `KoiTheme`（Orange/Red/Gold/Mixed）
  - 定义 `PetalDensity`（Sparse/Normal/Dense）
  - 实现状态循环切换方法
- **验证**：编译通过

### 任务 5.2 — 实现触控交互 ✅ 已完成
- **文件**：`entry/src/main/ets/pages/Index.ets` + `entry/src/main/ets/interaction/InteractionManager.ets`
- **内容**：
  - Index.ets 中 onTouch 检测点击事件，委托给 `InteractionManager.onTap()`
  - 点击屏幕：循环切换时钟样式（数字 → 极简 → 隐藏）
  - 交互状态通过 `RenderEngine.setClockStyle()` → `ClockLayer.setStyle()` 传递

### 任务 5.3 — 实现旋转表冠交互
- **文件**：`entry/src/main/ets/pages/Index.ets` + `entry/src/main/ets/interaction/InteractionManager.ets`
- **内容**：
  - Index.ets 中接入表冠事件（`@ohos.crown`），委托给 `InteractionManager`
  - 顺时针旋转：切换锦鲤主题色（`onRotaryClockwise` → `setKoiTheme`）
  - 逆时针旋转：切换花瓣密度（`onRotaryCounterClockwise` → `setPetalDensity`）
- **验证**：旋转表冠切换锦鲤/花瓣

### 任务 5.4 — 交互状态驱动渲染更新
- **文件**：`entry/src/main/ets/render/RenderEngine.ets` 及各 Layer
- **内容**：
  - `ClockLayer` 根据 `ClockStyle` 切换显示模式
  - `KoiLayer` 根据 `KoiTheme` 过滤显示的锦鲤颜色
  - `PetalLayer` 根据 `PetalDensity` 调整花瓣数量
- **验证**：交互后渲染立即响应

### 任务 5.5 — Phase 5 集成测试
- **验证**：
  - 点击切换时钟样式，3种模式循环
  - 旋转表冠切换锦鲤颜色
  - 旋转表冠切换花瓣密度
  - 交互响应即时，无延迟

**🔵 Phase 5 验收标准**：触控和表冠交互正常，渲染即时响应
